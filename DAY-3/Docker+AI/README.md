# Module 1 — Docker + AI 🐳

### First, understand what Docker actually does

Before AI can help you write Dockerfiles, you need the mental model:### Now use AI to generate a Dockerfile — and understand every line

Here's the prompt pattern for any Dockerfile task:

```
You are a senior DevOps engineer.

Write a production-ready Dockerfile for:
- Language/runtime: Python 3.11
- App: A Flask web app (app.py) with dependencies in requirements.txt
- Port: 8000
- Requirements: use slim base image, non-root user, layer caching optimized

Explain each instruction in a comment above it.
```

Paste that into your Day 1 CLI tool or directly into claude.ai. You'll get a fully commented, production-grade Dockerfile — and you'll understand every line because you asked for comments.
